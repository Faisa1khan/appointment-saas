# ADR 005: Internationalization (i18n)

## Context
Arrivo is designed for a global audience and requires robust support for multiple languages, localized date/time formatting, and bidirectional text (RTL) capabilities from day one.

## Decision
We adopted `next-intl` as our core internationalization library, paired with a strict "No hardcoded user-facing strings" policy governed by our Definition of Done.

## Why this approach was chosen
`next-intl` natively supports Next.js App Router and Server Components, providing a smooth developer experience without forcing client-side rendering for localized text. It leverages standard ICU message syntax, which gives us immense flexibility for pluralization and variables. By enforcing this deeply through engineering principles, we avoid the costly overhead of retrofitting translations later.

## Consequences
- Every new UI element requires dictionary entries in all supported languages (`en`, `hi`, etc.).
- Developers must use `useTranslations` (client) or `getTranslations` (server) exclusively for text content.
